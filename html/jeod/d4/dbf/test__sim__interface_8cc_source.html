139 /**
140  * Denote that error messages whose message code exactly matches the supplied
141  * code is a result of an condition being tested in some unit test. The
142  * registered 'allowed codes' are used two cases: fatal errors and non-fatal
143  * but severe errors, the latter defined in terms of the fatal threshold.
144  *
145  * The default exit status for program termination resulting from a call to
146  * MessageHandle::fail is one. When the message code matches one of the
147  * allowed codes the exit status will be zero. Since the JEOD unit test driver
148  * only looks at the program termination status, the allowed code mechanism
149  * provides a means for failure into success. What this means to you as a
150  * unit test developer is that you can now test the error tests in the mainline
151  * code. It also means you need to be careful in how you use this capability;
152  * maybe that successful failure is not as successful as you thought.
153  *
154  * Two things happen when a non-fatal but sufficiently serious MessageHandler
155  * call is made whose the message code matches one of the allowed codes:
156  * - The retrievable allowed_error_count is bumped. This provides a means
157  * for detecting in your unit test code that the an error condition was
158  * properly detected.
159  * - The test for a verboten code is bypassed. The allowed codes trump the
160  * verboten codes in case of conflict.
161  * \param[in] code The message code to be marked as allowed
162  */
163 void
164 TestSimInterface::add_allowed_code (
165  const std::string & code)
166 {
167  message_handler.add_allowed_code (code);
168 }

171 /**
172  * Denote that an error message whose message code partially matches the
173  * supplied code is bad news even if the error is not a fatal error.
174  * The test is of the first N characters of the incoming message code versus
175  * the verboten code, where N is the string length of the incoming code.
176  * The empty string as a verboten code will match all messages.
177  *
178  * The test will be terminated with non-zero exit status when an incoming
179  * non-fatal message
180  * - Has a severity level less than or equal to the failure_threshold,
181  * - Has a message code that does not match (exact match) any of the
182  * allowed codes, and
183  * - Has a message code that does match (initial match) one of the
184  * verboten codes.
185  * \param[in] code The message code to be marked as verboten
186  */
